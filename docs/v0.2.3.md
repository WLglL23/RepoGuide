# RepoGuide v2.0 → v2.3 阶段总结

## 概述

在 v1 Core SDK 建立稳定数据模型和门面后，v2.x 阶段的主要目标是：**将 RepoGuide 从一个纯 SDK 升级为具备 CLI、本地索引持久化和配置系统的正式工具**。整个过程严格遵循“新增不破坏、逐步迁移”原则，保持向后兼容。

---

## 一、v2.1：CLI 雏形与基础命令

### 做了什么
- 引入 Typer 框架，创建 `repoguide/cli/app.py`。
- 注册三个基础命令：
  - `repoguide version`：显示版本号。
  - `repoguide init`：创建 `.repoguide/` 目录结构。
  - `repoguide map .`：调用 `RepoGuide.map()` 输出格式化项目地图。
- 配置 `pyproject.toml` 中的 `[project.scripts]`，使 `repoguide` 命令可直接运行。
- 更新包级 `__init__.py` 以支持 `from repoguide import RepoGuide`。

### 新增/修改的文件
| 文件 | 说明 |
|------|------|
| `repoguide/cli/__init__.py` | CLI 模块入口 |
| `repoguide/cli/app.py` | CLI 命令定义：`version`, `map`, `init` |
| `repoguide/__init__.py` | 顶层导出 `RepoGuide` |
| `repoguide/core/__init__.py` | core 层统一导出 |
| `pyproject.toml` | 增加 Typer 依赖，配置脚本入口 |

### 设计目标与原因
- **统一的对外入口**：用户无需编写 Python 代码即可使用工具。
- **低耦合**：CLI 只负责参数解析和输出，核心逻辑完全委托给 `RepoGuide` 门面。
- **保留原有灵活性**：SDK 接口不变，旧脚本继续可用。

---

## 二、v2.2：本地索引持久化

### 做了什么
1. **补充模型序列化**
   - 为 `RepoSnapshot` 和 `ProjectMap` 添加 `from_dict()` 类方法。
   - 确保模型可从 JSON 完整还原。

2. **创建 storage 层**
   - 新增 `LocalIndexStore`，负责读写 `.repoguide/indexes/` 下的 `repo_snapshot.json` 和 `project_map.json`。
   - 使用 JSON 格式，避免过早引入 SQLite。

3. **扩展 Core Facade**
   - 新增 `RepoGuide.index(root_path)` 方法：扫描 → 生成快照 → 生成地图 → 保存索引。
   - 修改 `RepoGuide.map()` 行为：
     - 默认优先读取本地缓存。
     - 无缓存时临时扫描（不保存）。
     - 支持 `refresh=True` 强制重建并保存。

4. **升级 CLI**
   - 新增 `repoguide index .` 命令。
   - `repoguide map .` 支持 `--refresh` / `-r` 选项。
   - 为 `map` 命令添加缓存状态提示。

### 新增/修改的文件
| 文件 | 说明 |
|------|------|
| `repoguide/core/models/repo_snapshot.py` | 新增 `from_dict` |
| `repoguide/core/models/project_map.py` | 新增 `from_dict` |
| `repoguide/storage/__init__.py` | storage 包入口 |
| `repoguide/storage/local_index_store.py` | JSON 索引读写器 |
| `repoguide/core/repoguide.py` | 新增 `index` 方法，升级 `map` 支持缓存 |
| `repoguide/cli/app.py` | 新增 `index` 命令，`map` 支持 `--refresh` |

### 设计目标与原因
- **避免重复扫描**：通过本地缓存让后续 `map` 操作即时响应。
- **清晰的职责分离**：`index` 负责固化数据，`map` 负责快速查阅，不主动保存。
- **最小实现成本**：JSON 可读易调试，适合当前阶段，未来可平滑升级到 SQLite。

## 三、v2.3：配置系统集成

### 做了什么
1. **创建配置系统**
   - 定义配置数据模型：`RepoGuideConfig`, `ScanConfig`, `TestConfig`, `ProjectConfig`。
   - 抽取统一默认配置源 `default_config.py`，避免多处硬编码。
   - 实现 `ConfigLoader`：支持深度合并用户配置与默认配置，YAML 解析，异常回退。

2. **集成到扫描器**
   - 重构 `scan_repo()` 增加 `ignore_dirs` 和 `include_hidden_templates` 参数。
   - 内置默认值与外部参数合并，保留 `.gitignore` 支持（未删除原有逻辑）。

3. **集成到 SnapshotBuilder**
   - `SnapshotBuilder.build()` 接受可选 `RepoGuideConfig` 参数，并传递给 `scan_repo()`。
   - 项目名优先使用 `config.project.name`，否则回退到目录名。

4. **集成到 RepoGuide 门面**
   - `map()` / `index()` 自动调用 `ConfigLoader.load()` 加载配置。
   - 无配置文件时回退到内置默认值，无需强制 `init`。

5. **统一 CLI 初始化**
   - `repoguide init` 不再维护自己的配置字符串，改为调用 `ConfigLoader.create_default_config()`。

### 新增/修改的文件
| 文件 | 说明 |
|------|------|
| `repoguide/config/__init__.py` | 配置包入口 |
| `repoguide/config/config_model.py` | 配置数据类定义 |
| `repoguide/config/default_config.py` | 默认 YAML 配置内容 |
| `repoguide/config/config_loader.py` | 配置加载器（深度合并） |
| `repoguide/core/scanner/repo_scanner.py` | 增加参数化忽略目录/模板 |
| `repoguide/core/snapshot/snapshot_builder.py` | 透传配置，支持项目名 |
| `repoguide/core/repoguide.py` | 自动加载配置 |
| `repoguide/cli/app.py` | `init` 使用统一配置源 |
| `pyproject.toml` | 增加 `PyYAML` 依赖 |

### 设计目标与原因
- **可定制扫描行为**：用户可通过 `config.yml` 自定义忽略目录、白名单等。
- **单一配置源**：消除默认配置重复，未来维护只需修改一处。
- **平滑回退**：无配置文件时仍能正常工作，降低使用门槛。
- **为未来扩展奠基**：后续 `TestRunner`、`Indexer` 等模块可直接复用配置体系。

---

## 四、核心价值总结

经过 v2.0 到 v2.3 的建设，RepoGuide 具备了：

- ✅ **专业 CLI**：无需编程即可使用。
- ✅ **本地索引缓存**：大幅提升重复分析速度。
- ✅ **配置驱动**：灵活适配不同项目风格。
- ✅ **完整数据流**：`scan → classify → snapshot → project_map` 闭环。
- ✅ **向后兼容**：所有原有 Python API 和测试均保留有效。

这为后续高级功能（如结构化索引、LLM 问答、接口提取）提供了坚实的基础。
